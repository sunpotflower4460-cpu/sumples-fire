# UI/UX 改善サマリー — Fire Task

## 実施した改善内容

### P0: アクセシビリティの基本修正

#### 1. モーダルダイアログにフォーカストラップを追加
**ファイル**: `src/hooks/useFocusTrap.ts`（新規）, `src/App.tsx`

- `useFocusTrap` カスタムフックを新規作成
- モーダルが開いている間、Tab / Shift+Tab キーがダイアログ外に抜けないよう制御
- モーダルが開いたとき、最初のフォーカス可能な要素に自動フォーカス

**なぜ良くなったか**: スクリーンリーダーや全キーボード操作ユーザーが、ダイアログ背面のボタンを誤って操作する問題を防げる。`aria-modal="true"` の宣言と実際の動作が一致する。

#### 2. フィルターボタンに `aria-pressed` を追加
**ファイル**: `src/components/FireFilters.tsx`

- 各フィルターボタンに `aria-pressed={filter === item.value}` を追加
- グループ全体に `role="group"` を付与

**なぜ良くなったか**: スクリーンリーダーが「未燃焼フィルターが選択されています」と読み上げられるようになる。見た目の状態と支援技術の状態が一致する。

#### 3. 無効な `font-weight: 1000` を修正
**ファイル**: `src/comfortSettings.css`

- `font-weight: 1000` → `900` に変更（2箇所）
- `font-weight: 750` → `800` に変更（1箇所）

**なぜ良くなったか**: CSS 仕様の最大値 900 を超えた値はブラウザが丸める（または無視する）ため、意図した太さが確実に表示されるようになる。

### P1: 日常的な使いやすさの改善

#### 4. ランクプログレスバーに ARIA 属性を追加
**ファイル**: `src/components/FireStats.tsx`

- `role="progressbar"`, `aria-valuenow`, `aria-valuemin={0}`, `aria-valuemax={100}` を追加

**なぜ良くなったか**: スクリーンリーダーが「称号の進捗 72%」のように読み上げられる。視覚だけでなく音声でも達成度が伝わる。

#### 5. details 要素に展開・折りたたみインジケーターを追加
**ファイル**: `src/styles.css`

- `▸` アイコンを `summary::after` 擬似要素として追加
- 開いた状態では `rotate(90deg)` で `▾` の方向に変化
- `transition` でアニメーションを付与

**なぜ良くなったか**: 「詳しく書く」が折りたたみ可能な要素であることがひと目でわかる。押す前に中身があることを予告できる。

#### 6. トースト通知の表示時間を延長
**ファイル**: `src/hooks/useFireSeeds.ts`

- 2400ms → 3200ms に変更

**なぜ良くなったか**: 特に「ラスボス撃破！ +20炭になりました」のような長いメッセージを読み切る余裕が生まれる。

#### 7. タブ切り替え時に document.title を更新
**ファイル**: `src/App.tsx`

- アクティブタブが変わるたびに `document.title` を `今日 — Fire Task` のように更新

**なぜ良くなったか**: ブラウザのタブ・バーに現在の画面名が表示される。複数タブを開いているときの識別が容易になる。スクリーンリーダーもページ遷移として認識しやすくなる。

#### 8. idea-button にホバーフィードバックを追加
**ファイル**: `src/styles.css`

- `hover` 時に文字色が `#b24a1d`（ブランドカラー）に変化するトランジションを追加
- ボタンに最小限のパディングを追加して文字が枠に詰まらないよう調整

**なぜ良くなったか**: 空状態のアイデア候補リストがクリックできることが視覚的に伝わりやすくなる。

#### 9. matrix-badge / stage-row から非機能的な title 属性を削除
**ファイル**: `src/components/FireCard.tsx`

- `title="緊急度と重要度で自動分類しています"` を削除し、`aria-label` で代替
- `stage-row` の `title={stageDescriptions[seed.stage]}` を削除
- `stageDescriptions` の未使用インポートを削除

**なぜ良くなったか**: モバイル（タッチデバイス）では `title` ツールチップが表示されないため、情報が届いていなかった。`aria-label` を使うことでスクリーンリーダーには伝わり、視覚的にも分かりやすい表示が維持される。

## Before / After の要点

| 項目 | Before | After |
|------|--------|-------|
| モーダルでのTab操作 | ダイアログ外に抜けてしまう | ダイアログ内でループする |
| フィルター選択状態 | 視覚のみ（クラス） | `aria-pressed` で支援技術にも通知 |
| font-weight | 1000（仕様外） | 900（有効値） |
| ランク進捗 | 視覚バーのみ | `role="progressbar"` で値も読み上げ可 |
| details展開 | 展開インジケーターなし | `▸/▾` アニメーション付きインジケーター |
| トースト表示時間 | 2400ms | 3200ms |
| document.title | 常に「Fire Task」 | タブ名が付与（例：「今日 — Fire Task」） |
| アイデアボタン | ホバー変化なし | ブランドカラーに色変化 |
| matrix-badge title | モバイル非機能のtitle属性 | aria-labelで代替 |

## 今後の改善候補

### 高優先度
- **フィルターと今日タブの整合性**: 「燃焼済み」「すべて」フィルターを今日タブで選択しても `activeTasks` は burned を除外するため意味をなさない。フィルター表示を今日タブに適合した選択肢に絞るか、フィルタリングロジックを修正する
- **フォームエラー時のフォーカス管理**: バリデーションエラー発生時に `role="alert"` を持つエラー要素にフォーカスを移動する

### 中優先度
- **削除確認のカスタムUI**: 現状は `window.confirm()` 使用。カスタムの確認ダイアログにするとデザイン統一感が高まる
- **タスクなし→フィルターで空の場合の区別**: 「タスクがまだない」と「フィルターで該当なし」で異なる empty state を表示する
- **スケルトンローディング**: 将来的にAPI連携が入る場合、loading skeleton の土台を用意する

### 低優先度
- **マトリクスの4象限ラベルの視認性向上**: 現在の小さいラベルテキストをより目立たせる
- **タスクカードに作成日ラベルの改善**: 「追加 5月1日 12:00」の表示形式をより自然な日本語表現に
- **Storybook 導入**: 主要コンポーネントの状態（通常・is-burning・is-burned）を独立して確認できる環境整備

## 今回スコープ外にした項目

- `window.confirm()` による削除確認のカスタムUIへの置き換え（機能仕様変更を伴う）
- フィルターロジックとタブの整合性修正（機能仕様変更を伴う）
- Web Audio API のアクセシビリティ改善（`prefers-reduced-motion` に連動した音声制御）
- 色のみで意味を区別している箇所の完全解消（priority の色バッジにアイコンを追加等）
