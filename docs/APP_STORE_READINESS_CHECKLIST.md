# App Store 提出前チェックリスト

Sumples Fire を App Store に提出する前に確認する項目です。

このチェックリストは、提出ボタンを押す前の安全ゲートとして使います。

## 1. アプリ本体

- [ ] 起動直後にクラッシュしない
- [ ] iPhoneの縦画面で主要操作ができる
- [ ] メモの追加ができる
- [ ] メモの完了切り替えができる
- [ ] メモの削除前確認が出る
- [ ] localStorage保存が期待通り動く
- [ ] 空状態が不自然ではない
- [ ] 仮テキストや開発中の文言が残っていない
- [ ] App Storeスクリーンショットに使える画面になっている

## 2. 品質チェック

- [ ] npm run lint が成功する
- [ ] npm run test が成功する
- [ ] npm run typecheck が成功する
- [ ] npm run build が成功する
- [ ] GitHub Actions CI が成功する
- [ ] CodeRabbitなどでブロック指摘がない

## 3. アイコン / 表示

- [ ] 正式なApp IconをPNGセットで用意する
- [ ] Launch Screenを整える
- [ ] アプリ名が意図通り表示される
- [ ] iPhoneのノッチや下部バーにUIが被らない
- [ ] ダークモードで読みにくくならないか確認する

## 4. App Store Connect

ここは手動ゲートです。

- [ ] Bundle IDを作成する
- [ ] App Store Connectで新規アプリを作成する
- [ ] アプリ名を登録する
- [ ] カテゴリを選ぶ
- [ ] 年齢制限を設定する
- [ ] プライバシー情報を入力する
- [ ] スクリーンショットを登録する
- [ ] 説明文、キーワード、サポートURLを登録する
- [ ] 審査メモを必要に応じて書く

## 5. 署名 / 配布

ここは手動ゲートです。

- [ ] Apple Developer Teamが正しい
- [ ] Signingが成功している
- [ ] 実機ビルドが成功する
- [ ] Archiveが成功する
- [ ] TestFlightへアップロードできる
- [ ] TestFlightで実機確認する

## 6. 提出前の最終判断

- [ ] これは未完成のデモではなく、ユーザーが使えるアプリになっている
- [ ] 説明文と実際の機能が一致している
- [ ] 個人情報や外部送信がないことを確認した
- [ ] もし将来APIやDBを入れる場合、プライバシー表示を更新する
- [ ] 最終提出してよいと判断した

## 現時点の注意

現在の `public/icon.svg` は仮アイコンです。  
App Store提出前には正式アイコンへ差し替えてください。
