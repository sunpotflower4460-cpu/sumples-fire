# UI/UX Audit — Fire Task

## 調査概要

**アプリ**: Fire Task  
**スタック**: React 19 / TypeScript / Vite / Pure CSS / Web Audio API / localStorage  
**UIライブラリ**: なし（カスタムCSS）  
**画面構成**: 3タブ（今日 / 炭 / 使い方）＋ モーダルシート（タスク追加）

---

## 主要画面一覧

| 画面 | 説明 |
|------|------|
| 今日（today） | 未燃焼タスク一覧・マトリクスサマリー・炭ポイント・今日の火種カード |
| 炭（ash） | 燃焼済みタスク一覧・合計ポイント |
| 使い方（info） | アプリ説明・音設定・保存状況 |
| タスク追加シート | ボトムシートモーダル（緊急度・重要度・重さ選択） |

## 主要ユーザーフロー

1. **初回起動** → ヒーローセクションを見る → 「最初のタスクを書く」ボタンを押す
2. **タスク追加** → ＋ボタン or FAB → 名前入力 → 緊急度・重要度・重さ選択 → 保存
3. **タスク燃焼（Fire）** → 今日の火種カードまたはリストの「完了してFire」ボタン
4. **炭履歴確認** → 炭タブ → 燃焼済みタスク一覧
5. **設定確認** → 使い方タブ → 音ON/OFF・保存件数確認

## UI上の課題（優先度つき）

### P0: アクセシビリティの基本欠落

| # | 問題 | 対象ファイル | 影響 |
|---|------|------------|------|
| 1 | モーダルダイアログにフォーカストラップがない。`aria-modal="true"` が宣言されているが Tab キーでダイアログ背面のコンテンツにフォーカスが移動してしまう | `App.tsx` | スクリーンリーダー・キーボード操作ユーザーがダイアログを「抜けられない」感覚になる |
| 2 | フィルターボタンに `aria-pressed` がない。視覚的な選択状態はクラスで表現されているが、支援技術には選択状態が伝わらない | `FireFilters.tsx` | スクリーンリーダーユーザーが現在のフィルター状態を知れない |
| 3 | `font-weight: 1000` は CSS 仕様外（最大値は 900）。ブラウザによっては無視される | `comfortSettings.css` | テキストが意図した太さにならない可能性 |

### P1: 日常的な使いにくさ

| # | 問題 | 対象ファイル | 影響 |
|---|------|------------|------|
| 4 | ランクプログレスバーに `role="progressbar"` / `aria-valuenow` がない。視覚的なバーのみ | `FireStats.tsx` | スクリーンリーダーで進捗が読み上げられない |
| 5 | `<details>` の `summary` に展開・折りたたみ状態の視覚的インジケーターがない。webkit のマーカーを非表示にしているが代替のアイコンがない | `styles.css` | ユーザーが「詳しく書く」が押せることに気づきにくい |
| 6 | トースト通知の表示時間が 2400ms と短め。日本語テキストを読むのに余裕がない場合がある | `useFireSeeds.ts` | 通知を読み損ねる可能性 |
| 7 | タブ切り替え時に `document.title` が更新されない。常に "Fire Task" のまま | `App.tsx` | ブラウザ履歴・スクリーンリーダーで現在地が分からない |
| 8 | フィルターボタングループに `role="group"` がない。意味的なまとまりが弱い | `FireFilters.tsx` | スクリーンリーダーで文脈が失われる |
| 9 | `idea-button` にホバーのフィードバックがなく、インタラクティブな要素に見えにくい | `styles.css` | ユーザーがクリック可能と気づかない可能性 |

### P2: 品質向上・磨き込み

| # | 問題 | 対象ファイル | 影響 |
|---|------|------------|------|
| 10 | `matrix-badge` と `stage-row` に `title` 属性でツールチップを設定しているが、タッチデバイスでは機能しない | `FireCard.tsx` | モバイルユーザーに情報が届かない |
| 11 | フィルター「燃焼済み」「すべて」を今日タブで選択しても `activeTasks` の計算で burned タスクが除外されるため実質空になる（設計上の矛盾） | `App.tsx` | ユーザーが「バグ？」と感じる可能性（今回はスコープ外） |

---

## 影響が大きい改善ポイント

1. **フォーカストラップ**（P0）— キーボードユーザーの基本操作を保証する
2. **ARIA 属性の補完**（P0/P1）— スクリーンリーダー対応の基盤を整える
3. **details 展開インジケーター**（P1）— ユーザーが機能に気づきやすくなる
4. **document.title 更新**（P1）— ブラウザ標準の現在地表示を活用する

## 既存実装の良い点

- `aria-modal`, `aria-labelledby`, `aria-live`, `aria-hidden`, `aria-label` が多くの箇所で既に使われている
- フォーカスリング（`:focus-visible`）が全体に定義されている
- Escape キーでのモーダル閉鎖が実装済み
- Safe-area インセット対応が済んでいる
- prefers-reduced-motion のメディアクエリが適切に設定されている
- モバイル・デスクトップ両対応のレスポンシブ設計
